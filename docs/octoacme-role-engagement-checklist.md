# OctoAcme — Cross-functional Role Engagement Checklist

Use this checklist during planning and release readiness to make ownership and handoffs explicit across PM, PdM, Engineering, UX, BA, DevOps/SRE, and Support/Success.

## Initiation
- [ ] UX input captured for target users, journeys, and discovery needs
- [ ] BA input captured for business rules, process impacts, and constraints
- [ ] DevOps/SRE input captured for environment/reliability assumptions
- [ ] Support/Success input captured for top customer pain points and expected impact

## Planning
- [ ] Backlog items include UX artifacts/links where needed
- [ ] Requirements and assumptions are documented and traceable (BA + PdM)
- [ ] Reliability and deployment readiness requirements are documented (DevOps/SRE)
- [ ] Support enablement requirements are planned (docs/FAQ/training)

## Execution
- [ ] UX validation completed for implemented flows before sign-off
- [ ] Requirement changes recorded in backlog + decision log (BA + PM)
- [ ] Operational checks reviewed (monitoring, alerting, rollback) with DevOps/SRE
- [ ] Support/Success informed of known issues and response owners

## Release Readiness
- [ ] Cross-functional go/no-go includes PM, PdM, DevOps/SRE, and Support/Success
- [ ] Customer-facing release notes and support brief are complete
- [ ] Post-release monitoring ownership and escalation paths are confirmed

## Post-release Feedback Loop
- [ ] Support/Success summarizes early customer feedback within agreed window
- [ ] UX/PdM review adoption or usability signals and capture follow-up actions
- [ ] PM tracks follow-up actions to completion in backlog/retrospective
