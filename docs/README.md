# OctoAcme Project Management Overview

## Purpose
This README summarizes how OctoAcme delivers projects so new and current team members can quickly align on principles, roles, artifacts, and delivery rhythms.

## Core Principles
- **Customer-first:** prioritize user value and usability.
- **Iterative delivery:** deliver in frequent, small increments.
- **Clear ownership:** each project has a named Project Manager (PM) and Product Lead.
- **Data-informed decisions:** use measurable outcomes to guide prioritization and iteration.
- **Psychological safety:** promote feedback, learning, and continuous improvement.

## Project Lifecycle Stages
1. **Initiation:** clarify the problem, stakeholders, expected outcomes, and decision to proceed.
2. **Planning:** break work into a prioritized, actionable backlog and define milestones, risks, and dependencies.
3. **Execution & Tracking:** build, test, review progress, track work in the project board, and report metrics.
4. **Release & Deployment:** confirm readiness, deploy safely, validate outcomes, and communicate changes.
5. **Retrospective:** capture lessons learned and assign concrete improvement actions.

## Core Roles
- **Project Manager (PM):** coordinates execution, risks, timelines, and communications.
- **Product Manager (PdM):** owns what is built, prioritization, and success measurement.
- **Developers:** implement and test features.
- **QA/Testing:** validate acceptance and release readiness.
- **Stakeholders:** provide input, decisions, and approvals.

## Key Artifacts and Templates
- Project one-pager (charter)
- Prioritized backlog with acceptance criteria
- Risk register
- Release notes template
- Retrospective notes with action items

## Work Rhythms
- Daily or regular standups
- Weekly PM + PdM + delivery team syncs
- Demo/review at each sprint or milestone
- Monthly stakeholder updates and ad hoc escalations as needed
